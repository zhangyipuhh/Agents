# Soft Test Advanced · Systems Analyst Quick Reference (1 page)

> **National Exam framework** · Software Engineering + Information Systems + Informatization · Software project practical engineering issues and architecture · Practice layer

## 5 Major Knowledge Modules

| # | Module | Key Capabilities |
|---|---|---|
| 1 | System Planning (系统规划) | Feasibility analysis, project initiation, overall planning |
| 2 | Requirements Analysis (需求分析) | Business requirements, user requirements, functional / non-functional requirements, requirements modeling |
| 3 | System Design (系统设计) | Architecture design, detailed design, database design, interface design |
| 4 | Testing and Maintenance (测试与维护) | Test strategy, test cases, defect management, system maintenance |
| 5 | Informatization (信息化) | Information system integration, information security, e-commerce, e-government |

## Correspondence with Project Documents

| Module | Corresponding Project Document |
|---|---|
| System Planning | Pre-Sales Proposal, Project Initiation Report, Feasibility Study |
| Requirements Analysis | Requirements Specification, Requirements Specification Document |
| System Design | High-Level Design, Detailed Design, Database Design, Interface Design, UI Design |
| Testing and Maintenance | Test Plan, Test Report, Acceptance Report, Maintenance Manual |
| Informatization | Implementation & Deployment Plan, O&M Plan, Training Plan |

## Core Software Engineering Capabilities (interface with project documents)

- **Requirements analysis capability**: DFD, IDEF0, Use Case Diagram, Class Diagram, State Diagram
- **System design capability**: Architecture patterns (MVC / Microservices / Layered), database normalization, interface specifications
- **Testing capability**: Black-box / White-box / Gray-box; Unit / Integration / System / Acceptance testing
- **Maintenance capability**: Corrective / Adaptive / Perfective / Preventive maintenance
- **Informatization capability**: Information integration, security systems, standards and specifications

## Suite Usage Recommendations

- User asks "practical issues in software engineering" (performance / architecture / database / interfaces) → use the Systems Analyst practice layer
- User asks "how to write requirements completely" → Requirements Analysis module
- User asks "test coverage / test strategy" → Testing and Maintenance module
- User asks "implementation O&M / training" → Informatization module
