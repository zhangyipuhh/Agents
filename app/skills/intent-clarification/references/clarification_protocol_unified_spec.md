# Clarification Protocol Unified Specification

> Companion to `intent-clarification/SKILL.md`.

## Inquiry Workflow

```
Model/downstream skill needs to ask the user
  ↓
1. Identify the dimension (intent/data/environment/document_attr)
  ↓
2. Read <user_work_root>/.project/<project_id>/clarification_log.md
   ├─ Existing record for the same sub-item → show + ask "confirm/update"
   └─ No record → go to step 3
  ↓
3. Invoke query skill's read_doc.py to scan project materials
   ├─ Hit → cite file path + line number + original snippet + ask "use existing info? / supplement?"
   └─ Miss → ask directly
  ↓
4. Did the user answer "TBD" (待定)?
   ├─ Yes → ask again "stop the workflow / provide more details"
   └─ No → go to step 5
  ↓
5. Invoke manage_project_log.py append-clarification to append a record
  ↓
6. Return the answer to the calling skill
  ↓
7. After the workflow ends, invoke manage_project_log.py append-operation to write the main log
```

## Inquiry Wording Templates

For the specific wording of each dimension, see each skill's reference:

- Workflow clarification: `<user_work_root>/.project/<project_id>/intent.md` (if created)
- Actual location: `project-doc-query/references/intent.md`
- Data integrity: `project-doc-write/references/data_missing_section.md`, etc.
- Environment/Technology/Compliance: `project-doc-outline/references/tech_*.md` (10 files)
- Document attributes: `project-doc-write/references/role_signoff.md` / `doc_status.md`

## Path Resolution

- `<user_work_root>` (用户工作根) = parent directory of the project root
  - Example: project root `D:\项目文档\202410-C0008-...` → work root (工作根) `D:\项目文档\`
- `<project_id>` (项目号) = first 2 segments of the project root directory name
  - Example: `202410-C0008-...` → `202410-C0008`
- `.project directory` = `<user_work_root>/.project/`
- Project subdirectory = `<user_work_root>/.project/<project_id>/`

These path variables are auto-generated by `dump_paths.py` (loaded via PowerShell `. (ps1)`).
