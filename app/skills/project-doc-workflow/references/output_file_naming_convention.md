# Output File Naming Convention

> All documents generated by the project-doc suite uniformly follow this convention.

## Naming Format

### Project Directory (Official Document)

```
<project_root>/03_技术文档及评审/<subdir>/<type_name>.md
```

| Document Type | Subdirectory | File Name |
|---|---|---|
| Pre-Sales Proposal | 00_售前方案 | 售前方案.md |
| Requirements Specification | 02_需求分析 | 需求说明书.md |
| High-Level Design Specification | 03_概要设计 | 概要设计说明书.md |
| Detailed Design Specification | 03_概要设计 | 详细设计说明书.md |
| Implementation Plan | 01_实施方案 | 实施方案.md |
| Test Plan | 05_测试、验收 | 测试方案.md |
| Test Report | 05_测试、验收 | 测试报告.md |
| Acceptance Report | 05_测试、验收 or 07_验收结题 | 验收报告.md |
| Implementation & Deployment Plan | 06_实施部署及培训 | 实施部署方案.md |
| Training Plan | 06_实施部署及培训 | 培训方案.md |

### Change Log (Append)

```
<project_root>/06_变更及暂停/变更记录.md
```

Format:

```markdown
| Date | Project ID | Change Type | Document Name | Summary | Operator (System) |
|---|---|---|---|---|---|
| 2026-06-10 | 202410-C0008 | 新增 (Add) | 测试方案.md | 自动生成 V1.0 大纲与正文 | project-doc-skill |
```

### Intermediate Draft (AIAssistive\output\)

```
<work_root>/.aiassistive/output/<project_id>/<type_name>_draft.md
```

Example:

```
<work_root>/.aiassistive/output/<project_id>/<type_name>_draft.md
```

> Follows the AGENTS.md "Output Unified Aggregation" rule.
