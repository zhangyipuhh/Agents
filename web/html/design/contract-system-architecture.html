Multi-Agent System

合同智能体系统架构

基于 LangGraph 的多智能体协作平台，实现自然资源业务合同审批的智能化处理

🏗️
核心架构
🤖
HtAgent 主智能体
Host Agent
作为用户对话的主要入口，负责验证合同审批前置条件、调度子智能体、管理合同审批完整流程。
✓ 用户对话入口
✓ 流程调度协调
✓ 审批状态管理
✓
📄
DocAgent 文档智能体
Document Agent
负责拆解上传后的合同文件，提取文件内容和关键信息，为审批环节提供数据支持。
✓ 文件智能拆解
✓ 内容提取识别
✓ 关键信息提取
✅
ApprovalAgent 审批智能体
Approval Agent
负责合同的最后审批环节，从共享空间读取处理结果，执行审批决策并输出审批结果。
✓ 数据智能读取
✓ 审批决策执行
✓ 结果输出反馈

API 接口
POST
/api/contract/uploadfile
上传合同文件，系统自动处理并转换为图片格式，支持 PDF、Word、图片等多种格式
POST
/api/contract/chat
主智能体对话接口，处理用户与合同审批系统的交互，返回智能审批建议
POST
/api/contract/doc_chat
文档智能体对话接口，提取合同中的关键信息和条款内容
POST
/api/contract/store/value
获取共享存储空间中的数据，支持查询文件、审批结果等共享信息
💡
设计原则
1
上下文节省
子智能体（DocAgent、ApprovalAgent）每次执行完任务后会清空上下文空间，优化内存使用
2
数据持久化
通过 LangGraph Store 存储处理结果，实现跨会话数据访问和状态保持
3
直接数据读取
子智能体读取数据时直接从 Store 获取，无需通过上下文传递，提高效率
